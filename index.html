說明與注意事項
國家 + 城市自動對應邏輯：
1. 先輸入國家 / 地區，再輸入城市；例如國家輸入「韓國 / 南韓 / Korea / KR」，城市輸入「釜山 / Busan」。
2. 系統會先把常見中文城市別名轉成較穩定的英文查詢，例如 釜山→Busan、首爾→Seoul、東京→Tokyo、京都→Kyoto、台北→Taipei。
3. Geocoding API 回傳候選結果後，會用國家欄位加權排序，避免把釜山誤配到中國或其他地點。
4. 若城市仍有歧義，例如 Paris、Springfield、San José，可在「其他可能結果」下拉選單改選。
5. 天氣資料會依「城市座標 + 時區 + 年份範圍」快取；換日期區間不會重新打 API。
6. 遇到 429 時不會自動重試，會啟動本機冷卻，避免越按越被限流。
